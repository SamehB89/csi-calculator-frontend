üìä 1. What is Cost Breakdown Structure (CBS)?

Cost Breakdown Structure (CBS) is a comprehensive hierarchical classification of all project costs. While WBS focuses on "what we will build", CBS focuses on "how much everything will cost".

CBS Purpose

  • Budget Organization: Structure project budget logically
  • Cost Tracking: Monitor spending against baseline
  • Variance Analysis: Identify over/under budget areas
  • Historical Data: Build pricing database for future projects
  • Reporting: Generate financial reports at any level

4.1 Typical Code Structure

XX.YY.ZZ.NNN

XX = Major Division (01-99)
YY = Sub-Division (01-99)
ZZ = Work Category (01-99)
NNN = Sequential Item (001-999)

4.2 Extended Coding (with Location)

PRJ-BLD-FLR-DIV.SUB.CAT.NUM

PRJ = Project Code (HQ1)
BLD = Building (TWR-A, TWR-B)
FLR = Floor Level (B2, GF, TYP, RF)
DIV.SUB.CAT.NUM = CBS Code

üõ†Ô∏è 7. Implementation Guide

7.1 Setup Steps

  1. Define Structure: Determine levels and categories for your organization
  2. Create Coding: Design unique, logical coding system
  3. Document: Write CBS Manual with descriptions and rules
  4. Train Team: Ensure everyone uses correct codes
  5. Integrate: Link to schedule, accounting, and procurement
  6. Enforce: Make CBS mandatory for all transactions

7.2 Best Practices

  • Use unified coding across all company projects
  • Keep CBS simple - 4 levels usually suffice
  • Ensure CBS aligns with WBS for integration
  • Train accounting team on using correct codes
  • Build historical database for future estimating

‚ùå Common Mistakes

  • Creating too many levels (complexity kills adoption)
  • Inconsistent coding between projects
  • Not linking CBS to WBS and schedule
  • Allowing transactions without CBS codes
  • Not updating CBS when scope changes
